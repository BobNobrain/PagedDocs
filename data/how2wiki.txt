`[Раздел 1: Основные блоки и термины] Эти символы игнорируются
Страница разбивается на три части: заголовок, содержание и блок "см. также". Каждый блок
по-разному интерпретирует своё содержимое. Вся основная информация на странице содержится
в разделах вроде этого.

Структура этой секции: сначала идёт командная линия, содержащая заголовок; все последующие
линии - это содержимое раздела, вплоть до объявления следующего раздела или блока "см.
также".

КОМАНДНАЯ ЛИНИЯ - это строка, начинающаяся с символа '\`' (пробельные символы в начале
строки не учитываются). Если командный символ содержится в середине строки,то он 
обозначает КОМАНДНУЮ ПОСЛЕДОВАТЕЛЬНОСТЬ.
``Поскольку символ '`' является командным, то везде в тексте, кроме комментариев, если есть
``необходимость использовать его как обычный символ, то его необходимо экранировать
``обратным слэшем '\'

Одиночные символы перевода строки превращаются в пробелы. Чтобы заставить перевод строки
отобразиться на итоговой странице, поставьте два перевода строки подряд (как это уже
сделано несколько раз выше).

Вообще говоря, если вы поставите n символов перевода строки, то увидите на странице n-1 шт.

`[Раздел 2: Форматирование]
Форматирование текста осуществляется с помощью командных последовательностей. Вы можете
использовать их, чтобы создать `*жирный`, `_подчёркнутый`, `-перечёркнутый` текст, строчку
`.кода` или `/курсив`. То есть, сначала идёт командный символ, затем флаг форматирования,
затем - текст, который нужно форматировать, и завершается всё это ещё одним командным
символом. Если необходимо использовать командный символ внутри форматирования, вы всегда
можете экранировать его: `.Вот командный символ: \` - который не влияет на форматирование`.

`[Раздел 3: Блочное форматирование]
Поскольку просто форматировать текст было бы слишком скучно, я придумал такую весёлую вещь,
как блоки форматирования. Примером такого блока может служить список:
`{*
	`+ Список начинается (предыдущей) командной линией;
	`+ Звёздочка '*' указывает, что список неупорядоченный;
	`+ << Вот эти два символа обозначают начало нового элемента списка;
	`+ Табуляция перед каждым элементом здесь исключительно для красоты;
	`+ Список завершается закрывающей скобкой.
``(вот так:)
`}

Чтобы указать, что список должен быть упорядоченным, используйте '#' вместо '*'. Для 
краткости, совмещу этот пример с использованием вложенных списков:
`{#
	`+ Первый пункт
	`{*
		`+ Первый подпункт
		`+ Второй подпункт
	`}
	`+ Второй пункт
`}

Ещё пример такого блока: блок кода. Для кода указывается язык, на котором он написан
(см. ниже), чтобы правильно подсвечивать его синтаксис. Все символы табуляции и
перевода строк сохраняются внутри этого блока. Объявляется он так:

`(Java
public class Program extends AbstractUselessClass
{
	// Каждая строка переносится на страницу так, как она есть
	public static void main(String[] args)
	{
		System.out.println("Параметр языка определяет подсветку синтаксиса");
	}
}
`)

Следующий блок форматирования - спойлер:
`s|Заголовок спойлера
Внутри можно вкладывать любые блоки; например, вот вам `/листинг`
функции `.collapse()`:
`(Java
void collapse()
{
	this.reduceHeightTo(0);
	this.animate();
}
`)
Заканичивается блок спойлера вот так: 
(должен отсутствовать '|' после 's')
`s

`[Раздел 4: Ссылки]
Ссылки можно задавать двух типов: внутренние (на другую wiki-страницу) или внешние (на
сайт в этом вашем интернете). Первые открываются в текущей вкладке, вторые - в новой.
Ссылки объявляются следующим образом:

Вот так - ссылка на другую wiki-страницу: `{/main/GameCanvas.txt}Посмотрите на меня, я
- ссылка на GameCanvas!!!`; вот так - `<http://github.com/BobNobrain/PagedDocs/>ссыль` - 
на внешний сайт.

`[Раздел 5: Другие блоки]
На страницу ещё можно вставить:

`= - разделительную горизонтальную линию

Изображение (с подписью):
`i|/images/1.jpg|Форсим мем...

Таблицу:

`t|3|4
Первая колонка			|Вторая колонка		|Ну и третья, соответственно
Параметры после символа t:	|3 - столбцов,		|4 - строк
Новая ячейка начинается		|символом '|', 		|предварённым (хотя бы одним) символом табуляции
Перевод строки начинает		|новую строку таблицы.	|Работает экранирование: \|
После того, как были исчерпаны все заявленные (4) линии, начинается снова обычный текст.

Далее идёт блок "см. также".
`@ С этих символов он, собственно, начинается
Этот блок является неупорядоченным списком;
Каждая новая строка в нём создаёт отдельный пункт;
Можно использовать форматирование текста:
`{/main/0.txt}Куда относятся, кстати, и ссылки`;
`<http://google.com/>Ещё ссылка`;
Форматированный текст: `*b`, `/i`, `-s`, `_u`, `.code`;

Пустые строки игнорируются.
